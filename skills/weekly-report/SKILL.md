# 품질설계 에러 보고 스킬

## 사용 시기
- 사용자가 품질 에러 보고서, 불량 보고, 품질 이슈 보고서 작성을 요청할 때

## 보고서 양식 (반드시 아래 형식을 따를 것)

### [품질설계 에러 보고서]
- **보고 일시**: YYYY.MM.DD HH:MM
- **보고자**: (사용자 이름 또는 '작성자')
- **부서**: (소속 부서)

#### 1. 에러 개요
- **에러 코드**: (해당 시 기재)
- **에러 발생 일시**: YYYY.MM.DD HH:MM
- **발생 위치/공정**: (발생한 시스템, 설비, 공정명)
- **에러 유형**: (설계 오류 / 공정 불량 / 자재 불량 / 기타)

#### 2. 에러 상세 내용
- 에러 현상을 구체적으로 기술
- 영향 범위 및 수량 명시

#### 3. 원인 분석
- 직접 원인
- 근본 원인 (5-Why 또는 특성요인도 활용)

#### 4. 즉각 조치 사항
- 취한 임시 조치 내용
- 조치 일시 및 담당자

#### 5. 재발 방지 대책
- 단기 대책 (즉시 적용)
- 장기 대책 (시스템/공정 개선)
- 완료 예정일

#### 6. 첨부 자료
- 사진, 데이터, 도면 등 관련 자료 목록

## 규칙
- 각 섹션은 반드시 포함할 것
- 에러 코드 및 수치 데이터는 정확하게 기재
- 원인 분석은 추정이 아닌 근거 기반으로 작성
- 재발 방지 대책에는 반드시 완료 예정일 포함
